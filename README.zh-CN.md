# 人体功能哲学与判断
## 从状态理解到决策闭环

这是由 **Movetips 木梯原创提出** 的一套人体理解框架。

它不是诊断手册，不是动作清单，也不是零散技术的集合。  
它试图把人体理解重新拉回到一个更底层的起点：

**人体功能，就是身体被正常调用的能力。**

## 项目简介

本仓库用于公开介绍 Movetips 提出的人体功能哲学。

这里发布的内容，不是完整系统，也不是全量方法展开，  
而是对这套底层理解方式的公开说明。

## 核心观点

身体不是摆在那里等待被解释的。  
身体首先是拿来过生活的。

人要站、要走、要坐、要睡、要工作、要学习、要恢复、要运动、要照顾别人，也要面对每天真实的生活要求。

所以，更底层的问题不是只有：

- 这是什么病
- 这个动作哪里不对
- 该用什么技术

更底层的问题是：

**这个人的身体，现在还能不能被正常调用。**

## “调用”这条主线

**谁在调用？**  
任务、场景、生活需求。

**调用什么？**  
结构、能量、控制。

**怎么调用？**  
通过协作，而不是靠零散局部各自完成。

**调用失败了？**  
可能是没有、不会、不能。

**调用代价太高？**  
就会出现代偿、风险、不可持续。

**怎么让调用恢复正常？**  
通过评估、干预、训练。

“调用”这一个词，把整套理解框架都撑起来了。

## 这套框架为什么重要

这套框架想做的一件事，是让不同专业在面对同一个身体时，重新回到同一个对象上说话。

它可以帮助：

- 医生更容易把病名和真实功能状态接起来
- 治疗师更容易把评估和干预讲清楚
- 训练师不只看动作表面，而更看身体如何被组织起来
- 按摩师从局部感觉，进一步进入更完整的功能理解
